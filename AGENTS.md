# Agent Rules

Before any task:

1. Analyze the problem.
2. Create a short plan.
3. Implement changes.
4. Run validation.
5. Self-review.
6. Report results.

Required output:

## Analysis

## Plan

## Implementation

## Verification

## Changed Files

## Build Required

Rules:

* Never claim success without verification.
* Do not modify unrelated code.
* Prefer minimal changes.
* Use existing architecture.
* Ask before major refactoring.
* Verify before completion.

For UI work:

* Use UI/UX Pro Max.
* Mobile first.
* Premium visual design.
* Consistent spacing.
* Strong typography hierarchy.
* Accessibility.

Important:

Before testing any Flutter, Android, iOS, web frontend or mobile UI changes, always state whether a new build/deployment is required.

Use this format:

Build Required: Yes/No

If Yes:
Explain what must be rebuilt and why.
